05 · EDITABLE FIELDS

Anything a person can type into uses one of two models — no more. Everything else (segmented, toggle, stepper, select) is a control from §04, not a field. Both field models share one focus treatment: a blue 1px border plus a 3px halo.

1 · Inline edits in place

For primary content that is the thing — a card title or a stage name, edited right where it lives. Reads as plain text until you touch it, so the page stays calm. A quiet hover tint is the only cue that it's editable. Because there's no surrounding form, committing is explicit — Enter or ✓ saves, Esc or ✕ reverts. The ✓ / ✕ float just below the field so confirming never shifts the rest of the page.

For discrete data you fill in — a description, URL, email, comment, or the reject note. Always boxed, so it clearly reads as "type here." Comes in single-line and multi-line; identical styling, the textarea is just taller. How it commits depends on what the box is for — see Save model below.

Save model explicit Save / Cancel

Nothing on a card commits by ambient autosave. An inline title needs its ✓ / Enter; a comment needs its [Comment]. Boxed long fields follow the same rule: editing is a deliberate session you Save or Cancel. Save persists and collapses back to preview; Cancel reverts to the last saved text. A boxed field has three states.
